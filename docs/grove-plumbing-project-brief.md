# Grove Plumbing Redesign Demo — Project Brief

## Objective
Build a polished front-end demo for Grove Plumbing that modernises the company’s online presence and helps win the client by showing what the business could look like with:
- a much stronger visual presentation
- clear service and service area pages
- a realistic booking/request service flow
- stronger trust signals and clearer calls to action

This is **not** the final production site. It is a proposal-quality demo that should feel real enough for the owner to picture it live.

## Primary goals
1. Dramatic visual upgrade
2. Better trust and professionalism
3. More obvious lead-generation flow
4. A realistic preview of online booking
5. Stronger local SEO structure

## Audience
- Owner
- Potential staff
- Possibly customers if the demo link is shared

## Brand direction
The site should feel:
- warm
- local
- practical
- trustworthy

Avoid:
- cold SaaS styling
- generic AI-looking layouts
- corporate or luxury branding that feels too big-city
- overly flashy motion

## Visual references
- Acorn Heat
- Milani
- PJB Mechanical

Use those as inspiration for overall polish, trust, hierarchy, and service-business credibility — but keep Grove more local and grounded.

## Brand cues from Grove
Use the supplied assets where possible:
- Grove logo
- Grove van photos
- job photos
- family photo

These real images should do a lot of the trust-building work.

## Core messaging
The site should reinforce:
- local and family-oriented
- licensed and insured
- Red Seal certified
- experienced and established
- practical, friendly, and dependable
- easy to contact and easy to book

## Main conversion messages
The owner should feel:
- “I finally look like a real company.”
- “This would get me more calls.”
- “Customers could actually book easily.”
- “This makes me look bigger and more established.”
- “This would help me rank in more areas.”

## Tech stack
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui

## Build standard
Front-end only for now, but structure everything so it could later connect to:
- forms
- booking
- CRM
- review collection
- service request workflows

## Required pages
- Home
- About
- Services hub
- Service pages:
  - Plumbing Services
  - Drain Cleaning
  - Water Heater Repair & Installation
  - Heating Services
  - Gas Fitting
  - Emergency Plumbing
- Service Areas hub
- Service area pages:
  - Ladner
  - Tsawwassen
  - North Delta
- Book Service / Request Service
- Contact

## Required features
- Sticky call button
- Header CTA
- Strong homepage hero
- Services grid
- Testimonials slider
- FAQ sections
- embedded map section
- realistic multi-step booking UI
- repeated CTA sections throughout
- mobile-friendly layout

## Booking flow
The booking UI should feel realistic, but does not need a backend.

Suggested flow:
1. Select service
2. Choose preferred date
3. Choose a time window
4. Enter contact details
5. Show confirmation state

This can be framed as a request/booking hybrid:
“Request a visit and we’ll confirm your appointment shortly.”

## Content approach
Copy should:
- use Canadian spelling
- sound natural and local
- be professional without sounding corporate
- be clear and homeowner-friendly
- feel believable and specific

Do not write thin or generic copy.

## Trust signals to include
Use placeholders where exact details are unknown:
- years in business
- Google review rating/count
- licensed and insured
- Red Seal certified
- local/family-owned
- serving Ladner, Tsawwassen, and North Delta

## Design notes
- Use soft greens inspired by the logo
- Use off-white backgrounds instead of harsh white where helpful
- Keep contrast strong and readable
- Use real photos prominently
- Make CTAs visible without feeling pushy
- Give the booking UI special attention because it is one of the main “wow” features

## Demo mindset
This is a redesign proposal, a lead-generation website concept, and the beginning of a larger website + booking/CRM idea.

The result should feel:
- modern
- credible
- local
- built for leads
- easy to expand later
